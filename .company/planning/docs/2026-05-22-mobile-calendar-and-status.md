## 1. モバイルカレンダーの日付視認性改善

### 現状の問題
- 月表示：日付セルが小さく `text-[11px]` フォントで視認性が低い
- 週表示：日付部分が圧縮されてイベントと日付が判別しにくい
- 現場スタッフ（モバイルメイン）が日付を素早く確認できない

### 企画部門の見解
**必須改善。** 現場での使用頻度が最も高い画面なので、1秒で日付とイベントが把握できることが最低条件。

### 改善提案

#### 月表示
| 要素 | 現状 | 改善案 |
|-----|------|--------|
| 日付フォント | `text-[11px]` | `text-sm`（14px）+ `font-bold` |
| 日曜・土曜の色 | なし | 日曜=red、土曜=blue で視認性UP |
| 今日の強調 | インディゴ背景 | サイズをやや大きく + 背景色維持 |
| セル高さ | 固定 | `min-h-[5.5rem]`→`min-h-[6rem]` に拡大 |

#### 週表示
| 要素 | 現状 | 改善案 |
|-----|------|--------|
| 曜日ヘッダー | `text-[10px]` | `text-xs`（12px） |
| 日付 | `text-[11px]` | `text-sm font-black` |
| セル最小高さ | `min-h-[8.5rem]` | `min-h-[9rem]` |

### 工数見込み
- 半日〜1日（Tailwindクラス変更のみ、ロジック変更なし）

## 2. イベントステータス機能（準備中・入荷待ち）

### 現状
- `EventStatus = 'scheduled' | 'completed' | 'cancelled'` は型定義済み
- サイドバーに「準備中」「入荷待ち」ボタンが表示されているが、フィルター処理が未実装
- 新規イベント作成時に status フィールドが設定されない

### 企画部門の見解
**最優先で実装。** スプレッドシート時代の「準備状況」列に相当し、現場の日常業務の中心にある機能。これがないとアプリとして完成していない。

### 実装仕様

#### ステータス定義（拡張案）
```typescript
type EventStatus =
  | 'scheduled'    // 予定（デフォルト）
  | 'in_progress'  // 準備中
  | 'waiting'      // 入荷待ち
  | 'ready'        // 準備完了
  | 'completed'    // 終了
  | 'cancelled'    // キャンセル
```

#### UI表示
- イベントカード・カレンダーセルにステータスバッジを追加
- カレンダーイベントの左ボーダー色をステータスに応じて変化させる
- サイドバーフィルター：タップで該当ステータスのみ表示

#### ステータスバッジデザイン案
| ステータス | バッジ色 | ラベル |
|----------|---------|--------|
| scheduled | グレー | 予定 |
| in_progress | アンバー | 準備中 |
| waiting | オレンジ | 入荷待ち |
| ready | グリーン | 準備完了 |
| completed | スレート | 終了 |
| cancelled | レッド | キャンセル |

#### 実装箇所
1. `App.tsx` — サイドバーのステータスフィルター処理を実装
2. `App.tsx` — イベント編集モーダルにステータス選択UIを追加
3. カレンダーイベント表示にバッジを追加
